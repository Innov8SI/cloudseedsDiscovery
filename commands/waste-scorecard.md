# /waste-scorecard — Generate Final Scorecard

Compile everything from the session into three deliverable formats: Markdown, HTML, and DOCX.

Read all data from:
- `sessions/[slug]/session.json`
- `sessions/[slug]/waste-log.md`
- `sessions/[slug]/workflows/*.md`

---

## Step 1: Compile scorecard data

Aggregate:
- Client info, date, analyst
- All workflows: name, actors, systems, annual hours
- All wastes: full scored inventory
- Quick Wins and Big Bets lists
- Total savings opportunity
- Recommended roadmap (Phase 1/2/3)

**Roadmap logic:**
- Phase 1 (0–3 months): Quick Wins — payback < 6mo, impl cost < $40k
- Phase 2 (3–6 months): High-score wastes with moderate implementation
- Phase 3 (6–12 months): Big Bets — highest ROI but require significant build

## Step 4: Generate DOCX scorecard

Use the docx skill (see `/mnt/skills/public/docx/SKILL.md`).

Create `sessions/[slug]/scorecard/build-scorecard-docx.js` and run it with Node.js.

The DOCX should mirror the markdown structure with:
- Cover page: client name, date, analyst, gradient header using shading
- Page breaks between major sections
- Tables for: KPIs, workflow inventory, waste inventory (sorted by score), roadmap
- Color coding in waste table: red background for 🔴, yellow for 🟡, gray for ⚪
- Bold totals row in every table
- Footer: "Lean Waste Analysis | [client] | Confidential | Page N"

Primary color: `1B4F72` (dark blue). Accent: `F39C12` (orange).

After generating, validate with:
```bash
python /path/to/scripts/office/validate.py sessions/[slug]/scorecard/scorecard.docx
```
